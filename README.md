В проекте представлена реализация базовых структур данных на языке C с использованием хеш-функции FNV-1a для разрешения коллизий методом цепочек. 
Проект включает полноценную хеш-таблицу, множество с операциями объединения и пересечения, а также мультимножество с поддержкой кратности элементов. 
Код полностью документирован на русском языке, включает автоматические тесты всех операций, примеры практического применения (проверка уникальности, частотный анализ, системы прав доступа) и демонстрирует работу с коллизиями на маленьких таблицах.
Структуры данных поддерживают все базовые операции: вставку, удаление, поиск, обновление, а для множеств реализованы объединение, пересечение, разность и проверка подмножества. 
Мультимножество позволяет добавлять и удалять элементы с учётом кратности, получать общее количество элементов и количество уникальных значений. 
Компилируется стандартным компилятором C, не требует внешних зависимостей.
В проекте используется популярная Хеш-функция - FNV-1a (Fowler-Noll-Vo, вариант 1a).
Обоснование выбора Хеш-функции.
ВЫБОР ХЕШ-ФУНКЦИИ: FNV-1a (Fowler-Noll-Vo, вариант 1a)

FNV-1a — одна из лучших некриптографических хеш-функций.

ОБОСНОВАНИЕ ВЫБОРА:
1. Простота реализации — несколько операций XOR и умножения
2. Высокая скорость
3. Отличное распределение — минимальное количество коллизий
4. Используется в реальных проектах: DNS, HTTP-серверах, БД
5. Нет зависимости от длины строки (обрабатывает все байты)

Почему не djb2?
- FNV-1a лучше распределяет короткие строки
- FNV-1a использует простые арифметические операции

Параметры для 32-битной версии:
- offset_basis = 2166136261
- FNV_prime = 16777619

Хеш-функция - это функция, которая преобразует входные данные произвольного размера в фиксированное числовое значение или хеш. 
В общем случае, условно говоря, мы берем произвольные данные и преобразуем их в некоторое число в диапазоне от 0 до N-1, где N - размер таблицы.
Хорошая хеш-функция обладает несколькими ключевыми свойствами:
1) детерминированность: одинаковые входные данные всегда дают одинаковый результат
2) равномерность распределения: хеши должны равномерно распределяться по всему диапазону значений, минимизируя количество коллизий
3) скорость вычисления: функция должна работать быстро, иначе теряется весь смысл использования хеш-таблицы

Ключевой компонент хеш-функции - это ее алгоритм, в соответствии с которым вычисляется хеш. И таких алгоритмов может быть бесконечное множество. 
Основные простейшие реализации хеш-функций для строк:
1) Простейшая, но неэффективная реализация в виде суммирования ASCII-кодов символов. Проблема такой функции в том, что строки "abc" и "cab" дадут одинаковый хеш
2) Полиномиальная хеш-функция, которая учитывает позицию каждого символа.
3) Одной из популярных простейших и одновременно практических хеш-функций считается djb2, разработанная Дэниелом Бернштейном:
unsigned long hash_djb2(const char *str) {
    unsigned long hash = 5381;
    int c;
    while ((c = (unsigned char)*str++)) {
        hash = ((hash << 5) + hash) + c; /* hash * 33 + c */
    }
    return hash;
}
Принцип работы этой хеш-функции:
Она принимает данные произвольной длины (формально это строка, но фактически это может быть произвольный набор байтов)
Использует умножение на 33 (эффективно через сдвиг)
Возвращает число фиксированного размера
Стремится минимизировать коллизии (одинаковый хеш для разных данных)
Числа 5381 и 33 подобраны эмпирически и обеспечивают хорошее распределение битов. В итоге такая хеш-функция работает быстро и является простой в реализации. 

4) Популярная функция - FNV-1a (Fowler-Noll-Vo, вариант 1a).
Этот алгоритм FNV-1a особенно популярен в системном программировании, базах данных и сетевых приложениях, где скорость крайне важна.
uint32_t hash_fnv1a(const char *str) {
    uint32_t hash = 2166136261u;
 
    while (*str) {
        hash ^= (unsigned char)(*str++);
        hash *= 16777619;
    }
 
    return hash;
}
Эта функция также принимает некоторую строку и возвращает хеш-32-разрядное целое число без знака.
Вначале определяется переменная hash - это базис смещения FNV-1a (константа, специфичная для 32-битного FNV).
Это опытным путем подобранное простое число, которое служит начальным значением хеша.
Затем в цикле функция обрабатывает последовательно каждый символ строки.
Сначала выполняет операцию XOR с текущим значением хеша и перемещает указатель на следующий символ.
Затем умножает хеш на простое число FNV-1a (16777619).
Это простое число было выбрано из-за его математических свойств, которые помогают создать хорошее распределение

Ее преимущества: хорошее распределение, простота и высокая скорость.
Это практичная реализация, часто используемая по следующим причинам:
1) Она чрезвычайно быстрая (только XOR и умножение)
2) Отсутствие сложных зависимостей или выделения памяти
3) Хорошо работает с короткими строками (например, ключами словарей, именами переменных)
4) Часто используется в хеш-таблицах, компиляторах и интерпретаторах.
